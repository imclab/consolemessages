Chrome Console Messages
  • Content Security Policy
  • reflected-xss

If you see the following error in the console:

The 'reflected-xss' Content Security Policy directive has the invalid value value. Valid values are "allow", "filter", and "block".

then Chrome saw a Content-Security-Policy HTTP header that specified a value for the reflected-xss directive which couldn't be parsed. The directive accepts three valid values:

  • allow turns off the browser's active XSS protections
  • filter filters out potential XSS injections the browser detects
  • block blocks the entire page from loading if a potential XSS injection is detected

You can correct this error by specifying one of these valid options in the page's headers. For example, you might send the following header: Content-Security-Policy: reflected-xss block;.

Specification

Content Security Policy 1.1, section 4.12.4 - "reflected-xss"
